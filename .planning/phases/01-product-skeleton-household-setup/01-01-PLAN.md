<threat_model>
<trust_boundaries>
- 浏览器与 `apps/web` 之间交换登录表单输入与导航状态。
- `apps/web` 与 `apps/api` 之间交换登录凭证、会话 cookie 和家庭/物品 API 数据。

</trust_boundaries>

<threats>
- threat_id: T-01-01
  category: Spoofing
  component: 登录与会话基础
  disposition: mitigate
  mitigation_plan: 在 API 侧预留密码哈希校验和 `SessionMiddleware` 入口，约定使用 HttpOnly、SameSite=Lax 的 session cookie，并让受保护接口统一走登录依赖。
- threat_id: T-01-02
  category: Tampering
  component: 环境配置与本地服务编排
  disposition: mitigate
  mitigation_plan: 所有数据库、Redis、Web、API 配置都从环境变量读取，`.gitignore` 明确忽略 `.env*`、本地数据库文件和构建产物，避免把本地敏感配置提交进仓库。
- threat_id: T-01-03
  category: Denial of Service
  component: 开发环境启动链路
  disposition: mitigate
  mitigation_plan: 提供明确的健康检查、固定端口和最小启动脚本，避免后续执行阶段靠手工猜测服务状态。
</threats>
</threat_model>
