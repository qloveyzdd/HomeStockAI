<threat_model>
<trust_boundaries>
- `apps/api` 从购买记录、家庭画像和物品配置推导库存状态。
- Web 端通过 `GET /items/{id}/inventory-state` 和 `GET /items` 读取库存摘要。
- 保存购买记录时，API 会触发库存重算并覆盖当前物品快照。
</trust_boundaries>

<threats>
- threat_id: T-02-04
  category: Tampering
  component: 库存重算链路
  disposition: mitigate
  mitigation_plan: 库存快照只能由服务层重算生成，不能由客户端直接提交 `remaining_days`、`confidence_score` 之类字段。
- threat_id: T-02-05
  category: Repudiation
  component: 库存状态解释
  disposition: mitigate
  mitigation_plan: 每次重算都写入 `calc_reason` 和 `last_recalc_at`，至少留下记录数和 fallback 使用情况，避免后续无法解释结果来源。
- threat_id: T-02-06
  category: Information Disclosure
  component: 列表页轻量库存状态
  disposition: mitigate
  mitigation_plan: `/items` 只增加轻量库存字段，不暴露完整估算细节；详细数据只通过单物品接口获取。
</threats>
</threat_model>
