# Research Summary: 家补 AI（RefillWise）

## 最终建议

### 技术栈

- 前端：`Next.js 16` + `Tailwind CSS` + `shadcn/ui`
- 前端数据：`TanStack Query v5`
- 后端：`FastAPI 0.135.x`
- 数据层：`SQLAlchemy 2.0` + `Alembic 1.18`
- 存储：`PostgreSQL 17.x`
- 异步任务：`Redis 8` + `Celery 5.6`

### MVP 必须守住的能力

1. 家庭档案
2. 追踪物品
3. 购买记录
4. 库存估算
5. 价格快照
6. 推荐生成
7. 用户反馈校正
8. 低打扰提醒

### 真正的差异化

- 不是“哪个平台有活动”，而是“对我家现在值不值得买”
- 不是“家里有多少东西”，而是“该不该补、买几件”
- 不是一次性算完，而是通过用户反馈持续校正

## 架构结论

MVP 最合理的形态是：

- 一个 `web`
- 一个 `api`
- 一个 `worker`

都在同一仓库内，按领域模块分层，不做微服务。

## 最重要的风险

1. 冷启动输入负担过重
2. 库存估算不准却装得很准
3. 抓价不稳定影响主链路
4. 首页退化成优惠流
5. 提醒过多导致用户失去信任

## 对路线图的直接影响

- Phase 1 应先打基础模型和最小 Web/API 骨架
- Phase 2 必须优先做购买记录 + 库存估算
- Phase 3 再接价格快照与价格判断
- Phase 4 做推荐引擎和首页决策视图
- Phase 5 做反馈校正和提醒规则

## 一句话总结

这不是一个“先做很多 AI”的项目，而是一个“先把库存和价格判断做可信，再让 AI 负责解释”的项目。
